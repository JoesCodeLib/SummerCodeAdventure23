Prototype Chain

  • Prototype enables inheriting properties and methods from other objects. Every object = 1 prototype. If more than 2, null.
  • Don't really have to know

    • Destructuring

  • Instead of "const name = dog.name;", you can "const { name, dog } = dog;"
  • Can do same for arrays; const [ , , c] = arr; will assign the third array element to c.

    • ...spread

  • When you see "..." it means two objects merging
  • ex) obj2 ={...obj1, third = 'd', fourth = 'e'}

    • Optional Chaining

  • Use question mark "person?.name" to see if object is undefined.

    • Nullish Coalescing

  • Nullish Coalescing operator: ??
  • similar to || 'or' operator but only returns right hand side if left is null or undefined

    • Higher order function *Important*

  • Functions could be declarative or expression
  • function sayHi(message){ return "hi" + message }
  • const sayHi = function(message){return "hi" + message}
  • declare for global, express for local
  • Higher order function = 1. function who takes a function 2. function that returns a function
  • Could be expressed with arrow function => to make it easier and efficient. (using anony functions)

    • Closures

  • A function that relies on global variables or basically variables that are outside its own scope.
  • Does require more memory and power.
  • Three dots: Closure is function combined with its outer state or lexical environment.
  • Something about encapsulation but idk

    • Arrays in JS

  • "const arr = Array(100).fill(0)"   This fills an array with 100 things with all 0's
  • Can also add .map((_, i) => i +1) in-order to traverse and give all values their index
  • Set is like an array but can't be indexed and only hold unique values.
  • Could use set like "const unique = [...new Set(arr)];" in order to remove duplicates.
